# Git Content

Export and import Drupal content as Markdown files versioned in Git.

Each entity becomes a `.md` file with YAML frontmatter (metadata) and a
Markdown body, stored in a `content_export/` directory at the Drupal root.
This makes it straightforward to track content changes with version control,
sync between environments, or review diffs on any Git host.

## Requirements

- Drupal 10 or 11
- Core modules: `node`, `taxonomy`, `user`, `file`
- Optional core modules: `media`, `block_content`, `menu_link_content`
  (supported automatically when enabled)
- Composer libraries (installed automatically):
  - `league/html-to-markdown ^5.1`
  - `league/commonmark ^2.4`
- Optional PHP extension: `tidy` — pretty-prints exported HTML from
  `full_html` fields. The module works correctly without it.

## Installation

Install as you would any other Drupal module:

```bash
composer require drupal/git_content
drush en git_content
```

The `content_export/` directory is created automatically on first export.

To override the export directory, add to `settings.php`:

```php
$settings['git_content_export_dir'] = '../my-export';
```

Relative paths are resolved from the Drupal root.

## Usage

### Admin UI

Visit **Administration > Git Content Sync** (`/admin/git-content`).

The page shows a live preview of pending exports and imports, mirroring the
UX of Drupal's Configuration Synchronization page.

### Drush

```bash
# Export all content
drush git-content:export        # alias: gce

# Export a specific entity type
drush git-content:export nodes
drush git-content:export taxonomy
drush git-content:export media
drush git-content:export blocks
drush git-content:export menus
drush git-content:export files
drush git-content:export users

# Import all content
drush git-content:import        # alias: gci
```

## Features

- **Multilingual** — default translations are exported/imported before
  non-default translations to preserve the original language.
- **Checksum-based skip** — unchanged files are not re-exported or
  re-imported, keeping diffs clean.
- **Import order** — files, users, taxonomy, and media are imported before
  nodes; entity references are resolved correctly.
- **Author preserved** — `uid` is stored as a username and resolved on import
  across environments.
- **Taxonomy term IDs preserved** where possible so Views filters keep working.
- **Menu link URLs** are converted from environment-specific
  `entity:node/{id}` to portable `internal:/{alias}` format.
- **Auto-generated files excluded** — image style derivatives, oEmbed
  thumbnails, and media icons are skipped (regenerated by Drupal on render).

## File structure

```
content_export/
  site.yaml                     ← site-wide metadata (languages, front page)
  content/
    article/
      en/my-article.md
      es/mi-articulo.md
  taxonomy/
    tags/
      en/drupal.md
  media/
    image/
      en/my-image.md
  blocks/
    basic/
      en/my-block.md
  menus/
    main/
      en/home.md
  files/
    my-file.md
  users/
    editor.md
```

## File format

Each file has YAML frontmatter followed by a Markdown body:

```markdown
---
uuid: a1b2c3d4-e5f6-7890-ab12-cdef01234567
type: article
lang: en
status: published

title: My article
slug: my-article

created: 2024-01-15
changed: 2024-03-10
author: editor

path: /my-article

taxonomy:
  tags:
    - Drupal

checksum: 0beec7b5ea3f0fdbc95d0dd47f3c5bc275da8a33
---

Body content in **Markdown**.
```

| Field | Description |
|---|---|
| `uuid` | Entity UUID — primary key for matching on re-import |
| `type` | Entity bundle |
| `lang` | Language code |
| `status` | `published` / `draft` / `active` / `blocked` / `permanent` |
| `author` | Account name of the entity owner |
| `translation_of` | UUID of the default translation (non-default only) |
| `checksum` | SHA1 of frontmatter + body; skips import when unchanged |

## Submodules

### git_content_layout

Exports and imports **Layout Builder** per-entity overrides.

- Each layout section and its components are stored as structured YAML.
- Inline block references are stored by UUID for portability.
- Requires `layout_builder`.

### git_content_paragraphs

Exports and imports **Paragraphs** fields inline within their parent entity.

- Paragraph data is embedded directly in the parent `.md` file.
- Existing paragraphs are updated in place on re-import.
- Requires `paragraphs`.

## Important notes

- **Binary files** are not exported. Only file metadata (`uri`, `filename`,
  `mime`) is stored. Manage binaries separately (e.g. `git-lfs`, `rsync`).
- **Passwords** are never exported. User 1 is never overwritten on import.
- **Comments** are not exported or imported.

## Optional: tidy extension

The PHP `tidy` extension pretty-prints HTML exported from `full_html` text
fields. Without it, HTML is exported correctly but on a single line.

**DDEV** — add to `.ddev/config.yaml`:

```yaml
webimage_extra_packages:
  - php8.3-tidy
```

**Lando** — add to `.lando.yml`:

```yaml
services:
  appserver:
    build:
      - apt-get install -y php8.3-tidy
```
